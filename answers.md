## - Вопрос 1: объясни мне REST
Ответ:
REST (Representational State Transfer) — это парадигма проектирования API, основанная на принципах клиент-серверной архитектуры и не являющаяся протоколом передачи данных. Основные принципы REST включают:

1. Клиент-серверная архитектура: Сервер предоставляет ресурсы, которые клиент может получать или изменять.
2. Stateless: Сервер не хранит информацию о сессии клиента и должен обрабатывать каждый запрос независимо от предыдущих.
3. Кэширование: Каждый ответ сервера должен содержать метку, позволяющую определить, можно ли его кэшировать для уменьшения нагрузки на системы и сетевого трафика.
4. Единообразие интерфейса: Все ресурсы представлены в виде репрезентаций (например, JSON), которые могут быть получены или изменены с помощью стандартных HTTP методов (GET, POST, PUT, DELETE).
5. Layered System: Сервер может быть слоем системы, где каждый слой выполняет свою функцию и не знает о существовании других слоев.
6. Code on Demand (необязательно): Сервер может передать код клиенту для выполнения на стороне клиента.

REST-подход не всегда требует использования HTTP или JSON, хотя они часто используются в реализациях RESTful API. Важные характеристики RESTfull-сервисов включают:

- Уровни зрелости: Определены как Уровень 0 (без REST-ограничений), Уровень 1 (с одним ограничением), Уровень 2 (с двумя ограничениями) и Уровень 3 (с тремя или более ограничениями).
- Масштабируемость: Благодаря Stateless, сервисы могут быть легко масштабированы.
- Простота поддержки: Stateless упрощает логику на стороне сервера и позволяет использовать кэширование.

REST не является протоколом передачи данных, а концепция парадигмы проектирования API. Он определяет принципы и ограничения для создания интерфейсов между клиентами и серверами, обеспечивая гибкость и масштабируемость.

## - Вопрос 2: какие коды http ответов относятся к успешному ответу?
Ответ:
Коды HTTP ответов, которые относятся к успешному ответу:

- 200 OK - Успех
- 201 Created - Ресурс успешно создан
- 202 Accepted - Запрос принят, но обработка еще не завершена

## - Вопрос 3: дай мне определение JSON
Ответ:
JSON (JavaScript Object Notation) — текстовый формат обмена данными, основанный на JavaScript. Но при этом формат независим от JS и может использоваться в любом языке программирования.

## - Вопрос 4: в чем отличие уровня 1 и уровня 2 в модели зрелости REST
Ответ:

Уровень 1 в модели зрелости REST не использует правильные HTTP-глаголы для создания или удаления ресурсов, как это делается на уровне 2. На уровне 2 используются глаголы GET, POST, PUT и DELETE для соответствующих операций. Например, чтобы создать спикера, используется метод POST/api/speaker, а чтобы получить информацию о спикере — метод GET/api/speaker.

На уровне 1 эти операции могут быть выполнены любыми HTTP-глаголами, что не всегда соответствует спецификации HTTP-протокола и может приводить к неправильному использованию ресурсов.

Таким образом, уровень 2 в модели зрелости REST более строго придерживается правилами использования HTTP-глаголов для создания и удаления ресурсов.
