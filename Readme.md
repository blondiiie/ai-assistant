# Личный AI-ассистент (RAG по своим заметкам)

Telegram-бот, который отвечает на вопросы **только по твоим заметкам и файлам**
(Obsidian-vault и/или локальные папки) по принципу RAG. Киллер-фича —
**отсутствие галлюцинаций**: бот отвечает лишь на то, что реально есть в
твоих материалах, иначе честно говорит, что информации нет.

> Проект вырос из «корпоративного AI-ассистента». После пивота целевой
> пользователь — не компания, а любой человек (например, студент), который хочет
> искать выжимки информации по собственным файлам: «второй мозг» с поиском по
> смыслу.

## Возможности

- **Семантический поиск** по заметкам (не по ключевым словам, а по смыслу).
- **Анти-галлюцинация** (3 уровня): порог релевантности检索 → grounded-генерация
  с проверкой покрытия ответа контекстом → корректная атрибуция источника.
- **Источники в ответе**: `[Источник: путь · раздел]` — всегда можно проверить.
- **Автосинхронизация**: правки заметок подхватываются автоматически
  (ленивый mtime-рескан перед ответом) + команды `/rescan`, `/status`.
- **Локальность и приватность**: всё работает на твоём ноутбуке, LLM и
  эмбеддинги — локально через Ollama. Ничего не уходит наружу.
- **Поддержка форматов**: Markdown (с обработкой Obsidian-синтаксиса), TXT, PDF, DOCX.

## Как это работает (RAG-цикл)

```
Вопрос → векторизация вопроса → гибридный поиск top-k чанков (вектор + лексика)
       → порог релевантности → grounded-генерация (локальная LLM)
       → проверка опоры на контекст → ответ + источник
```

**Наполнение базы (ингест):** папка-источник сканируется → каждый файл парсится
→ режется на чанки (~512 токенов) с метаданными (файл, раздел, страница) →
каждый чанк векторизуется (`nomic-embed-text`) → сохраняется в PostgreSQL+pgvector.
Заголовок заметки добавляется в чанк (contextual chunking) — это улучшает
семантическое сопоставление.

## Стек

- **Backend:** Python 3.12, FastAPI, aiogram 3 (Telegram), SQLAlchemy 2 (async).
- **БД:** PostgreSQL 16 + pgvector (HNSW + GIN индексы), в Docker.
- **LLM / эмбеддинги:** локальная Ollama (`qwen2.5:3b-instruct`, `nomic-embed-text`).
- **RAG-пайплайн:** собственный, без LangChain.

## Про Obsidian (важное уточнение)

Ассистент работает **с папкой-хранилищем (vault)**, а не через Obsidian API или
плагины. Технически это чтение локальной директории с `.md`-файлами, но парсер
**учитывает Obsidian-диалект Markdown**:

- разворачивает `[[wikilinks]]` в чистый текст;
- убирает YAML-frontmatter;
- заголовки `#/##` становятся метаданными раздела;
- игнорирует служебные папки `.obsidian/`, `.trash/` и вложения.

Так же индексируются обычные папки с PDF/DOCX/TXT. Источник настраивается в `.env`
(`SOURCE_DIRS`).

## Запуск

```bash
make up            # поднять PostgreSQL+pgvector
make initdb        # создать схему (один раз)
make serve         # FastAPI (админ-загрузка /documents) — опционально
make bot           # Telegram-бот (стартовый скан источников + обработка вопросов)
```

В `.env` указать: `TELEGRAM_BOT_TOKEN`, `ALLOWED_CHAT_IDS`, `SOURCE_DIRS`.

## Команды бота

- текст — задать вопрос по заметкам;
- `/start`, `/help` — справка;
- `/upload` — загрузить файл вручную (PDF/DOCX/TXT);
- `/rescan` — принудительно переиндексировать источники;
- `/status` — статистика (источники, число документов/чанков).

## Ограничения (текущая версия)

- **Модель 3B вариативна**: иногда добавляет смежные «учебные» термины или ложно
  отказывается отвечать. Для более стабильных ответов можно переключиться на
  `qwen2.5:7b-instruct` (`LLM_MODEL` в `.env`) — потянет M5/16 ГБ.
- Бот работает, только пока запущен на ноутбуке (не 24/7).
- **Multi-turn / уточнения по предыдущему вопросу** — в планах (post-MVP).
