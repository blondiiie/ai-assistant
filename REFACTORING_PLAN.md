## ЧАСТЬ 1. ИЗВЕСТНЫЕ БАГИ (требуют исправления)

### 1.1. 🔴 BUG: Заглушка на вопросах «что такое JSON» / «что такое HATEOAS»

**Симптом:** На вопрос «что такое JSON» или «что такое HATEOAS» бот выдаёт
заглушку об отсутствии информации в заметках, хотя соответствующие заметки
**существуют** в Obsidian-vault и проиндексированы.

**Гипотезы причины (по вероятности):**

1. **Retrieval не находит чанк.** Термин «JSON»/«HATEOAS» — короткий,
   эмбеддинг может быть слабым (не хватает контекста вокруг термина в чанке).
   `sim_threshold=0.35` может отсечь релевантный чанк, если формулировка
   вопроса («что такое X») далека от формулировки в заметке («X — это...»).
   - **Диагностика:** `uv run python -m scripts.diagnose "что такое JSON"` —
     посмотреть top-k чанков и их hybrid-score. Если релевантный чанк есть, но
     score < 0.35 — проблема в пороге.

2. **Grounding валит валидный ответ.** Короткий термин («JSON — формат обмена
   данными») может не набрать `grounding_score ≥ 0.5` из-за низкой
   trigram_overlap (термин в заметке развёрнут, ответ краток). Хотя код должен
   пропускать короткие ответы без `min_kept_sentences`, единый `grounding_score`
   всё ещё может отсечь.
   - **Диагностика:** в `diagnose.py` посмотреть raw-ответ и grounding-компоненты.

3. **Чанкование разрезает определение.** Если определение «JSON — ...» попадает
   на границу чанка, контекст обрывается и модель не может дать полный ответ.
   - **Диагностика:** проверить в БД чанки, содержащие «JSON».

**План исправления:**
1. Воспроизвести через `scripts/diagnose "что такое JSON"` и `"что такое HATEOAS"`.
2. Если retrieval находит чанк, но grounding отсекает — снизить
   `grounding_threshold` для коротких ответов или добавить exception для
   терминологических вопросов.
3. Если retrieval не находит — снизить `SIM_THRESHOLD` до 0.30 или добавить
   keyword-boost для точных совпадений термина.
4. Добавить регрессионный тест на эти вопросы.

что нет роста (утечек) и пик ≤ зелёной зоны.

> после рестарта дают одинаковый (на попытке 0) результат.
